# __Краткое описание__  
&nbsp;

__CSSJ__ - компактная библиотека содержащая в себе основные классы для создания отзывчивых и адаптируемых web-страниц. CSS классы генерируются по HTML разметке в зависимости от class аттрибута. Основные генерируемые классы описаны в разделе [Классы и свойства](#классы-и-свойства). Для установки и настройки пакета используются стандартные npm команды:

```
npm i cssj
npm explore cssj -- npm run init  
```

После выполнения указанных команд в папке проекта будет создан файл конфигурации с именем - ___cssj.config.json___. _(Подробнее про конфигурацию см. в [следующем разделе](#конфигурация))_

После того как HTML будет написан, либо в процессе его написания чтобы сгенерировать CSS классы и при необходимости объединить их с пользовательскими классами используется команда:

```
npm explore cssj -- npm run build
```
либо

```
npm explore cssj -- npm run build watch
```

Второй вариант позволяет отслеживать изменения в уже созданных файлах и делать ребилд собранного CSS файла.

# __Конфигурация__
&nbsp;

Стандартный файл конфигурации выглядит следующим образом

```json
{
	"BUILD_PATH": "{project-path/build}",
	"HTML_PATHES": [
		"{project-path}"
	],
	"CSS_PATHES": [],
	"IGNORE_PATHES": [],
	"SET_CLASSES": {},
	"MERGE_SAME_SELECTORS": false
}
```

#### __BUILD_PATH__ 

__Тип:__  ( _String_ )

Путь к папке(или файлу) билда, если указан каталог то файл по умолчанию будет иметь имя __app.css__
&nbsp;

---
#### __HTML_PATHES__ 

__Тип:__  ( _String[]_ ) 

Массив путей к папкам или HTML файлам проекта
&nbsp;

---
#### __CSS_PATHES__ 

__Тип:__  ( _String[] / object[]_ )
 
Массив путей к каталогам или CSS файлам, _(если необходимо их объединить с итоговым build файлом)._
Файлы CSS будут собраны в порядке следования путей, те что расположены в конце массива имеют более высокий приоритет. В случае если указан путь к каталогу, то файлы в каталоге будут собраны в порядке их обнаружения _( поэтому если необходима точная последовательность лучше указывать путь к фалам)_. Также приоритет может быть указан самостоятельно если вместо строкого пути использовать объект вида:

```json
{ 
    "path":"path", 
    "priority":10 
}
```
&nbsp;

---
#### __IGNORE_PATHES__ 

__Тип:__  ( _String[]_ ) 

Массив путей которые будут игнорироваться при сканировании html и сборке css файлов. Каталог ___node_modules___ всегда игнорируется по умолчанию.
&nbsp;

---
#### __SET_CLASSES__ 

__Тип:__  (_Object_) 

Объект содержащий в котором можно переопределить стандартные настройки CSS классов - название, единицы, приоритет, точность. _Например:_ Стандартный класс __m__ отвечает за свойство margin и измеряется в единицах rem, имеет точность 1. Чтобы поменять эти значения нужно добавить следующее свойство:

```json
{
	"SET_CLASSES": {
        "m":{
            "rename":"margin", //новое имя класса
            "unit": "px", //меняем единицы на px
            "precision": 0, //кол-во цифр после запятой = 0
            "priority":100 //более высокий приоритет
        }
    },
}
```

Раньше при использовании класса __m-15__ мы бы получили следующее:

```css
.m-15{
    margin:1.5rem
}
```

Теперь необходимо использовать класс __margin-15__:

```css
.margin-15{
    margin:15px;
}
```

Однако не все классы можно настраивать полностью _(более подробно о каждом классе см. Раздел [Классы и свойства](#классы-и-свойства))_
&nbsp;

---
#### __MERGE_SAME_SELECTORS__ 

__Тип:__  ( _Boolean_ ) 

Свойство указывающее, делать ли слияние одинаковых селекторов. Слияние в одном файле происходит согласно специфичности CSS _(чем ниже в файле расположен селектор тем он приоритетнее, но не всегда)._ Слияние из разных файлов происходит по их приоритету _(см. [CSS_PATHES](#css_pathes))_. Для at-rules существует свой приоритет который в данный момент нельзя поменять или настроить.
__Стандартные селекторы__ по умолчанию имеют приоритет - __0__. Чем больше приоритет тем ниже будут располагаться at-rules и селекторы относительно друг друга.

Приоритет at-rules и селекторов следующий:

- __@namespace__ _(приоритет: -11)_
- __@import__ _(приоритет: -10)_
- __@charset__ _(приоритет: -9)_
- __@font-face__ _(приоритет: -8)_
- __@font-feature-values__ _(приоритет: -7)_

        Внутренние at-rules для @font-feature-values _(@character-variant, @styleset, @stylistic, @ornaments, @annotation, @swash)_ тот же приоритет

- __@font-palette-values__ _(приоритет: -6)_
- __@page__ _(приоритет: -5)_
- __@color-profile__ _(приоритет: -4)_
- __@container__ _(приоритет: -3)_
- __@property__ _(приоритет: -2)_
- __@starting-style__ _(приоритет: -1)_
- __@keyframes__ _(приоритет: 1)_
- __@counter-style__ _(приоритет: 2)_
- __@supports__ _(приоритет: 3)_
- __@scope__ _(приоритет: 4)_
- __@layer__ _(приоритет: 5)_
- __@media__ _(приоритет: 6)_

К некоторым at-rules при слиянии применяются не стандартные правила, например @font-face для проверки повторяемого значения использует ___"src"___ свойство внутри font-face. 
Для текущей версии CSSJ поведение при слиянии не настраивается но в дальнейшем такая возможность может появиться.

# __Классы и свойства__

Большинство классов имеют следующую структуру:

```
(префикс-)(класс)(-important)(_псевдокласс)-(значение)(ед.измерения)
```

&nbsp;

## __Префиксы__

#### __md-__ _(mobile device)_ префикс для медиа-запроса:

```css
@media(max-width:1023px) and (max-height:1023px)
```
_Пример:_
```html
<div class="md-fsz-15"></div>
```
_Результат в css:_
```css
@media(max-width:1023px) and (max-height:1023px){
    .md-fsz-15{
        font-size:1.5rem;
    }
}
```

---

#### __mdl-__ _(mobile device landscape)_ префикс для медиа-запроса:

```css
@media(max-width:1023px) and (max-height:1023px) and (orientation:landscape)
```
_Пример:_
```html
<div class="mdl-fsz-15"></div>
```
_Результат в css:_
```css
@media(max-width:1023px) and (max-height:1023px) and (orientation:landscape){
    .mdl-fsz-15{
        font-size:1.5rem;
    }
}
```

---

#### __mdp-__ _(mobile device portrait)_ префикс для медиа-запроса:

```css

```


## __important__

Чтобы указать значения как important к классу добавляется постфикс __-i__

_Пример:_
```html
<div class="m-i-10"></div>
```
_Результат в css:_
```css
.m-i-10{
    margin: 1rem !important;
}
```
&nbsp;

## __Псевдоклассы__

На данный момент поддерживается лишь малая часть частоиспользуемых псевдоклассов css, а именно:

- __hover__ _(постфикс: _h)_
- __focus__ _(постфикс: _f)_
- __blur__ _(постфикс: _b)_
- __active__ _(постфикс: _a)_
- __disabled__ _(постфикс: _d)_
- __visited__ _(постфикс: _v)_

Не исключено что с обновлениями библиотеки этот список пополнится, и некоторые постфиксы могут претерпеть изменения.
Для использования псевдокласса постфик указывается после класса или [__-i__](#important) постфикса

_Пример:_
```html
<div class="m_h-10"></div>
```
_Результат в css:_
```css
.m_h-10:hover{
    margin: 1rem;
}
```
_Пример с __important__:_
```html
<div class="m-i_h-10"></div>
```
_Результат в css:_
```css
.m-i_h-10:hover{
    margin: 1rem !important;
}
```
&nbsp;

## __Единицы измерения__


Единицы измерения как и их точность уже по умолчанию установлена для всех классов имеющих численные характеристики_(подробнее про каждый в разделе [Классы](#классы))_.
Тем не менее если есть необходимость изменить стандартные единицы измерения есть два пути:
1. Изменить настройки класса в конфигурации _(см. раздел [__Конфигурация__](#конфигурация))_
2. Применить указание единицы к значению в классе:

```html
<div class="m-10px"></div>
```
_Результат в css:_
```css
.m-i_h-10:hover{
    margin: 1rem !important;
}
```
Оба варианта работают только в том, случае если для класса разрешено использовать свойство __unit__ при настройке в __SET_CLASSES__, иначе класс не будет добавлен в итоговый сss файл
